2. 내용의 문서 구조를 생각한다.
	1) 마크업 전의 HTML 문서를 확인한다
		→ 콘텐츠 부분이 HTML 택로 마크업되어 있지 않기 때문에 뺵빽한 형태가 나온다
		→ 정확하게 마크업되어 있지 않은 HTML 문서는 컴퓨터에서 보면 분석하기 힘든 문자 모음으로 정보로서 이용하기 어려움
		→ HTML 태그를 사용해 콘텐츠 내용에 적절하게 의미를 부여하고 컴퓨터에서도 구조를 해석하기 쉽게 해야 함
	2) 원고에서 문서 구조를 읽는다
		→ HTML로 표현 가능한 범위에서 그 문서의 정보 구조(문서 구조)를 읽고 적절한 요소 적용
		(1) 표제를 찾는다.
			→ 각 그룹의 표제에 해당하는 부분을 '표제 요소'로 지정
			→ 표제를 나타내는 h 요소는 큰 제목, 중간 제목, 소 제목 식으로 표제 레벨에 따라 h1 ~ h6까지 6단계가 있음
		(2) 항목 리스트를 찾는다
		(3) 문장 모음을 찾는다
			→ 표제와 항목 부분이 결정된 후 텍스트나 본문 등과 같은 문장 콘텐츠를 '단락'으로 마크업
		(4) 기타 정보 구조를 찾는다
3. HTML 태그로 마크업하기
	1) 표제를 마크업한다
		→ 표제가 설정된 곳은 글자가 크고 굵어지며 줄이 바뀌어 앞뒤로 공백이 생김
		→ 최상위 표제 h1은 필수적으로 페이지에 1개가 있음
		→ h2 이하는 문서 구조에 따라 적당히 사용할 수 있지만, 중간 레벨을 건너뛰거나 레벨의 상하 관계를 바꾸는 것은 원칙적으로 불가능
		→ h1 ~ h6의 표제 요소로 만들어진 트리 구조는 그대로 HTML의 문서 구조의 골격이 됨
	2) 단락을 마크업한다
	3) 항목을 마크업한다
		→ ul(ol) 요소와 li 요소는 2개로 하나를 이루기 때문에 각각 단독으로 사용할 수 없음
		→ li 요소 안에 다시 요소를 넣을 수 있음
	4) 기술 목록(정의 목록)을 마크업한다
		→ 기술 목록: 항목과 그 설명이 한 세트로 된 리스트 구조
		→ 전체를 <dl>~</dl>로 감싸고 항목을 <dt>~</dt>, 그 설명을 <dd>~</dd>로 마크업
		→ <dt>와 <dd>가 한 세트가 되어 하나의 정보 항목이 되는 것 → <dt>만 사용하거나 <dd>만 사용하는 것은 불가능
		→ 하나의 <dt>에 여러 <dd>가 매달리는 구조는 가능
	5) 정보의 그룹화
		→ 정보 그룹화를 위해 기본적으로 div 요소 사용
		→ 목적
			(1) 정보를 정리할 때마다 소스를 그룹화하고 거기에 이름을 붙여 소스의 가독성을 높인다.
			(2) 레이아웃이나 디자인의 구현을 위해 필요한 범위를 지정한다.
		→ 요소 이름은 id 속성을 사용해 지정
	6) div를 section으로 바꾼다
		→ HTML5에서는 '표제와 그에 따른 콘텐츠'같이 문서의 골격을 이루는 중요한 그룹에 새로운 전용 요소인 section 요소 사용 가능
		→ 섹션 요소는 기본적으로 제2계층 이하 표제에 대해서만 사용 가능
	7) 불필요한 구분선 삭제
4. 기억해 두어야 할 마크업 규칙
	1) 요소의 구조와 부모 자식 관계 
		→ 요소의 관계를 바르게 유지 
	2) 콘텐츠 모델과 요소 분류
		→ 콘텐츠 모델: 어느 요소 안에 어떤 요소를 넣을 수 있는지를 정한 규칙
		→ 단순한 개념: HTML5 이전 버전에서 정해 놓은 '블록 요소/인라인 요소'라는 분류 방법으로 생각하는 방법
			→ 블록 요소: 표제, 단락, 항목, 표, 등과 같은 문서 구조의 골조를 구성하는 요소군 => 정보의 그릇
			→ HTML의 내용물: 텍스트 데이터나 이미지 등과 같은 콘텐츠
			→ 인라인 요소: 내용물이 되는 콘텐츠에 의미를 부여하는 용도로 사용되는 요소 => 텍스트 데이터와 같은 취급을 받는 텍스트 레벨의 요소
			→ 블록 안에 인라인을 넣을 수는 있지만 거꾸로는 허용되지 않는다
